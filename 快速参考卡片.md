# 博客部署快速参考卡片

## 🎯 一句话方案
GitHub 存源码 → Actions 自动构建 → 发布到 GitHub Pages + Vercel → 腾讯云 CDN 加速国内访问

---

## ⚡ 5 分钟写作流程

### 方式 1：在线编辑（零安装）
```
1. 打开 github.com/你的仓库
2. 按 . 键进入在线编辑器（或访问 github.dev/你的仓库）
3. 在 source/_posts/ 创建 Markdown 文件
4. 左侧 Git 面板提交
5. 等待 1-2 分钟自动部署
```

### 方式 2：本地写作（功能最全）
```bash
git pull                       # 同步
hexo new "文章标题"            # 创建
# 编辑 source/_posts/文章标题.md
hexo server                    # 预览（可选）
git add . && git commit -m "新文章" && git push
```

## 📦 初次部署核心步骤

### 第 1 步：创建 Hexo 项目
```bash
npm install -g hexo-cli
hexo init my-blog && cd my-blog && npm install
```

### 第 2 步：配置核心文件
1. **_config.yml**（网站配置）
   - url: `https://你的域名.com`
   - 标题、描述、作者等

2. **.github/workflows/deploy.yml**（自动部署）
   ```yaml
   - uses: peaceiris/actions-gh-pages@v3
     with:
       github_token: ${{ secrets.GITHUB_TOKEN }}
       publish_dir: ./public
       cname: 你的域名.com
   ```

### 第 3 步：推送到 GitHub
```bash
git init && git add . && git commit -m "init"
git remote add origin https://github.com/用户名/仓库名.git
git push -u origin main
```

### 第 4 步：启用 GitHub Pages
- Settings → Pages → Source: `gh-pages` 分支
- Custom domain: 填写域名

### 第 5 步：连接 Vercel
- 导入 GitHub 仓库
- Framework: Hexo
- Build: `hexo generate`
- Output: `public`

### 第 6 步：配置 DNS
```
# 国内线路
CNAME @ → [腾讯云CDN的CNAME]

# 国外线路（可选）
CNAME @ → cname.vercel-dns.com
```

### 第 7 步：配置腾讯云 CDN
- 源站地址：`你的用户名.github.io` 或 `你的项目.vercel.app`
- 开启 HTTPS
- 配置缓存规则

## 🔧 常用命令速查

### Hexo 命令
```bash
hexo new "标题"        # 新建文章
hexo new page "关于"   # 新建页面
hexo generate          # 生成静态文件（或 hexo g）
hexo server            # 本地预览（或 hexo s）
hexo clean             # 清理缓存
```

### Git 命令
```bash
git pull               # 拉取更新
git status             # 查看状态
git add .              # 添加所有更改
git commit -m "说明"   # 提交
git push               # 推送
```

### 一键发布
```bash
hexo clean && hexo g && git add . && git commit -m "update" && git push
```

## 🐛 故障排查速查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 推送后未部署 | Actions 失败 | 查看 Actions 日志 |
| 样式丢失 | URL 配置错误 | 检查 `_config.yml` 的 `url` 和 `root` |
| 404 错误 | Pages 未生效 | 确认 gh-pages 分支存在 |
| 域名无法访问 | DNS 未生效 | 等待 DNS 生效（可能需要几小时）|
| 国内访问慢 | 未走 CDN | 检查 CDN 配置和 DNS 解析 |

```
                ↑
        推送代码到 GitHub
                ↑
        本地/在线编辑 Markdown
```
